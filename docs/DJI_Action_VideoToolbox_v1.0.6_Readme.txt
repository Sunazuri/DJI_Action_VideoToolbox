📌 1. このソフトは何をするツールか
============================================================

DJI_Action_VideoToolbox_v1.0.6 は、DJI Osmo Action / DJI Action 系カメラで撮影した動画を扱いやすくするための Windows 用 GUI ツールです。

主な機能は2つです。

  1️⃣ 動画連結 / VideoConcat
      複数の .mp4 / .mkv 動画を、指定した順番で1本の動画へ連結します。

  2️⃣ DJI Action LUT / HEVC エンコード
      DJI の D-Log M 動画へ「DJI OSMO Action 4 D-Log M to Rec.709 V1.cube」などの .cube LUT を適用し、HEVC / NVENC で書き出します。

v1.0.6 では、1920×1080 解像度・Windows表示スケール125%付近でも、ボタンや下部ステータスが隠れにくいようにレイアウト耐性を強化しています。


============================================================
🧰 2. 事前に必要なもの
============================================================

このツール単体では動画変換そのものは行いません。
内部で FFmpeg / ffprobe を呼び出します。

必要なもの:

  ✅ Windows 11 推奨
  ✅ ffmpeg.exe
  ✅ ffprobe.exe
  ✅ LUTを使う場合は .cube ファイル
     例: DJI OSMO Action 4 D-Log M to Rec.709 V1.cube
  ✅ NVIDIA GPU搭載PCで高速処理する場合は、NVENC対応GPUと対応ドライバ

ffmpeg.exe / ffprobe.exe は、以下のどちらかで指定できます。

  方法A: 画面上部の「共通設定」で ffmpeg.exe / ffprobe.exe を直接指定する
  方法B: ffmpeg / ffprobe を PATH に通し、ソフト側の欄は空欄のまま使う

確実性を重視するなら、方法Aの「直接指定」を推奨します。

🎞️ 5. 動画連結 / VideoConcat の使い方
============================================================

動画連結タブでは、複数の .mp4 / .mkv を1本へまとめられます。

------------------------------------------------------------
5-1. 入力動画を追加する 📥
------------------------------------------------------------

追加方法は3つあります。

  ✅ 「ファイル追加」ボタン
  ✅ 「フォルダ追加」ボタン
  ✅ リスト欄へ動画ファイルまたはフォルダをドラッグ＆ドロップ

対応拡張子:

  ・.mp4
  ・.mkv

フォルダを追加した場合、「フォルダ再帰追加」にチェックが入っていれば、サブフォルダ内の動画も対象になります。

------------------------------------------------------------
5-2. 連結順を変更する 🔃
------------------------------------------------------------

動画はリストの上から下の順番で連結されます。

順番を変えるには、対象動画を選択して以下のボタンを使います。

  ・上へ
  ・下へ

複数選択にも対応しています。

------------------------------------------------------------
5-3. 不要な動画を削除する 🧹
------------------------------------------------------------

  ・選択削除
      選択した動画だけをリストから削除します。

  ・全クリア
      連結リストを空にします。

※元の動画ファイル自体は削除されません。

------------------------------------------------------------
5-4. 出力先を指定する 💾
------------------------------------------------------------

「出力フォルダ」に保存先を指定します。

「出力ファイル名」には、完成動画のファイル名を入力します。

例:

  joined_video.mp4
  joined_video.mkv
  2026-05-01_撮影連結.mp4

便利機能:

  ✅ .mp4 だけを追加した場合、出力拡張子は .mp4 へ自動調整されます。
  ✅ .mkv だけを追加した場合、出力拡張子は .mkv へ自動調整されます。
  ✅ .mp4 と .mkv が混在した場合のみ警告が表示されます。

------------------------------------------------------------
5-5. 連結方式を選ぶ ⚙️
------------------------------------------------------------

連結方式は2種類あります。

  1️⃣ 高速・無劣化連結（-c copy / 同一仕様の動画向け）

      映像と音声を再エンコードせず、ほぼコピーのまま連結します。
      最も高速で、画質劣化も基本的にありません。

      向いている条件:
        ・同じカメラで連続撮影した動画
        ・解像度、フレームレート、映像コーデック、音声仕様が揃っている動画
        ・同じ .mp4 同士、または同じ .mkv 同士

      注意:
        仕様が異なる動画を無理に -c copy 連結すると、音ズレ・途中停止・再生不可の原因になります。

  2️⃣ 互換重視連結（H.264 + AAC へ再エンコード）

      映像を H.264、音声を AAC へ変換しながら連結します。
      -c copy より時間はかかりますが、仕様の違う動画をまとめたい場合に有利です。

      向いている条件:
        ・異なる機材の動画が混じっている
        ・再生互換性を優先したい
        ・SNSや一般的なプレイヤーで再生しやすくしたい

------------------------------------------------------------
5-6. 連結オプション ✅
------------------------------------------------------------

  ✅ 既存ファイルを上書き
      同名の出力ファイルが存在する場合に上書きします。
      OFFの場合、同名ファイルがあると開始前に停止します。

  ✅ 完了後に出力フォルダを開く
      連結完了後、保存先フォルダを自動で開きます。

  ✅ .mp4 はWeb最適化
      .mp4 出力時に -movflags +faststart を付けます。
      Web配信やネットワーク越し再生で、再生開始が早くなる場合があります。
      ファイルサイズは基本的に大きく変わりません。

  ✅ フォルダ再帰追加
      フォルダ追加時に、サブフォルダ内の .mp4 / .mkv も探します。

------------------------------------------------------------
5-7. 連結開始 ▶️
------------------------------------------------------------

設定が終わったら「連結開始」を押します。

処理中はログ欄に進行状況が表示されます。
途中で止めたい場合は「キャンセル」を押します。

------------------------------------------------------------
5-8. 連結タブの便利機能 💡
------------------------------------------------------------

  🧩 コマンド表示
      現在の設定から生成される FFmpeg コマンドを表示し、クリップボードへコピーします。
      GUIでうまくいかない時の検証や、コマンドラインで再実行したい時に便利です。

  📝 ログ保存
      連結ログをテキストとして保存できます。
      失敗原因の確認や、後日の検証に有効です。

  📁 出力フォルダを開く
      手動で出力先フォルダを開けます。

  🎯 自動拡張子調整
      .mp4のみ、または.mkvのみなら出力ファイル名の拡張子を自動で合わせます。
      混在時だけ警告を出す設計です。


============================================================
🚁 6. DJI Action LUT / HEVC エンコード の使い方
============================================================

このタブでは、DJIのD-Log M動画へLUTを適用したり、HEVCへ再エンコードしたりできます。

対応入力拡張子:

  ・.mp4
  ・.mov
  ・.mkv
  ・.m4v

------------------------------------------------------------
6-1. LUT .cube を指定する 🎨
------------------------------------------------------------

「LUT .cube」欄に、使用する .cube ファイルを指定します。

例:

  DJI OSMO Action 4 D-Log M to Rec.709 V1.cube

指定方法:

  ✅ 参照ボタンで選択
  ✅ ファイルを欄へドラッグ＆ドロップ

重要:

  LUTは映像の色を変えるフィルタです。
  そのため、LUTを動画に焼き込む場合、映像の再エンコードは必須です。

  -c:v copy のまま LUT だけを適用することはできません。

------------------------------------------------------------
6-2. 出力先を指定する 📁
------------------------------------------------------------

「出力先」に保存フォルダを指定します。

空欄の場合:

  入力動画と同じフォルダへ出力されます。

これはGUI上では気付きにくい便利機能です。
「元動画と同じ場所に変換後ファイルを作りたい」場合は、あえて出力先を空欄にしても使えます。

------------------------------------------------------------
6-3. 出力接尾辞を指定する 🏷️
------------------------------------------------------------

「出力接尾辞」は、元ファイル名の後ろに付く文字列です。

例:

  入力:
    DJI_0001.MP4

  出力接尾辞:
    _DLogM_to_Rec709_HEVC

  出力:
    DJI_0001_DLogM_to_Rec709_HEVC.mp4

処理モードを変更すると、既定の接尾辞が自動的に切り替わる場合があります。

------------------------------------------------------------
6-4. 入力動画を追加する 📥
------------------------------------------------------------

追加方法は3つあります。

  ✅ ファイル追加
  ✅ フォルダ追加
  ✅ リスト欄へファイルまたはフォルダをドラッグ＆ドロップ

「フォルダ再帰追加」がONの場合、フォルダ内だけでなくサブフォルダも検索します。

------------------------------------------------------------
6-5. エンコード基本オプション ✅
------------------------------------------------------------

  ✅ LUT適用 ON
      指定した .cube LUT を適用します。
      標準モードではON/OFFを切り替え可能です。

  ✅ HEVC NVENC
      NVIDIA GPUのハードウェアエンコーダで HEVC 出力します。
      NVIDIA GPU搭載PCでは高速です。
      NVIDIA以外のGPU環境については後述します。

  ✅ 10-bit Main10
      10-bit HEVCとして出力します。
      D-Log M素材の階調を扱う場合はON推奨です。

  ✅ 音声コピー
      音声を再エンコードせずコピーします。
      音声劣化を避けたい場合はON推奨です。

  ✅ Web用に最適化
      .mp4 に -movflags +faststart を付けます。
      Web配信やネットワーク再生で、再生開始が早くなる場合があります。

  ✅ 既存出力を上書き
      同名ファイルがある場合に上書きします。
      OFFの場合は同名出力をスキップまたは停止します。

  ✅ 室内向け微補正
      標準モード時のみ使う軽い補正です。
      内部的には、LUTの後に軽い eq 補正を追加します。
      必要な時だけ使ってください。

  ✅ フォルダ再帰追加
      フォルダ追加時に、サブフォルダ内の動画も対象にします。

------------------------------------------------------------
6-6. 数値設定の意味 🔢
------------------------------------------------------------

  並列数
      同時に何本の動画を処理するかを指定します。
      初期値はCPU論理プロセッサ数から安全側に自動計算されます。
      AMD Ryzen AI 9 HX 370 のような 12C/24T 級では、初期値3を想定した設計です。

      注意:
        NVENCは速いですが、同時処理数を上げすぎるとGPU・ストレージ・発熱・VRAMの競合が起きます。
        安定重視なら 1〜3 を推奨します。

  CQ
      画質基準値です。
      小さいほど高画質・大容量になりやすく、大きいほど低容量・低画質になりやすいです。
      目安は 18〜22 です。

  Bフレーム
      圧縮効率に関わります。
      多いほど効率は上がりますが、処理負荷や互換性に影響する場合があります。

  Lookahead
      先読み解析です。
      画質・圧縮効率に効きますが、速度は落ちます。

  AQ強度
      Adaptive Quantization の強さです。
      細部や暗部の見え方に影響します。

------------------------------------------------------------
6-7. 処理モードの詳細 🎛️
------------------------------------------------------------

v1.0.6 には5つの処理モードがあります。

  1️⃣ 標準：LUT + HEVC/NVENC エンコード

      通常使用向けの基本モードです。
      LUT適用、HEVC/NVENC、Main10、AQ、Lookahead、Bフレームなどを使い、画質と容量のバランスを取ります。

      向いている用途:
        ・D-Log M素材をRec.709へ変換したい
        ・最終保存用に画質と容量を両立したい
        ・NVIDIA GPUで安定したHEVC変換をしたい

      特徴:
        ・NVENC preset p7 / tune hq 系の品質寄り設定
        ・Bフレーム、Lookahead、AQを使用
        ・LUT補間方式は選択可能

  2️⃣ LUTのみ適用：品質優先

      LUTを主目的にした品質優先モードです。
      「LUT以外の余計な補正をなるべく加えず、cubeの色変換を丁寧に焼き込む」用途に向いています。

      特徴:
        ・LUT適用は必ずON
        ・音声コピーはON
        ・室内向け微補正はOFF
        ・LUT補間方式を選択可能
        ・既定では tetrahedral 高品質 が選ばれます

      注意:
        このモードでも映像は再エンコードされます。
        LUTを焼き込む以上、完全無変換コピーは不可能です。

  3️⃣ 高速LUT適用：trilinear + NVENC p1

      速度を重視しつつ、画質も極端には落としたくない場合の推奨高速モードです。

      特徴:
        ・LUT補間方式は trilinear に固定
        ・NVENC preset p1
        ・Bフレーム 0
        ・Lookahead 0
        ・AQ OFF
        ・音声コピー

      向いている用途:
        ・大量のD-Log M動画へ素早くLUTを当てたい
        ・まず確認用を作りたい
        ・速度と画質のバランスを取りたい

  4️⃣ 最速LUT適用：nearest + NVENC p1

      速度を最優先するモードです。

      特徴:
        ・LUT補間方式は nearest に固定
        ・NVENC preset p1
        ・Bフレーム 0
        ・Lookahead 0
        ・AQ OFF
        ・音声コピー

      注意:
        nearest は最速ですが、色のつながりや階調が粗く見える可能性があります。
        最終保存よりも、確認用・仮書き出し向けです。

  5️⃣ HEVC変換のみ：LUTなし

      LUTを使わず、HEVC変換だけを行うモードです。

      向いている用途:
        ・すでに色補正済みの動画を圧縮したい
        ・単純にHEVCへ変換したい
        ・LUTファイルを使わない処理をしたい

------------------------------------------------------------
6-8. LUT補間方式の詳細 🎨
------------------------------------------------------------

LUT補間方式は、LUTの色変換をどのように補間するかを決めます。

  ✅ trilinear 推奨
      速度と画質のバランスが良い方式です。
      通常はこれを選べば十分です。

  ✅ nearest 最速
      最も軽い方式です。
      ただし、階調や色のつながりが粗くなる可能性があります。
      確認用や速度最優先向けです。

  ✅ tetrahedral 高品質
      高品質寄りの方式です。
      色の滑らかさを重視する場合に向いています。
      速度は trilinear より遅くなる場合があります。

注意:

  高速LUT適用モードでは trilinear 固定です。
  最速LUT適用モードでは nearest 固定です。
  そのため、該当モードではLUT補間方式の選択欄が無効化されます。

------------------------------------------------------------
6-9. 自動走査 🔍
------------------------------------------------------------

「自動走査」ボタンは、選択中の動画を ffprobe で解析し、以下の値を自動入力します。

  ・CQ
  ・Bフレーム
  ・Lookahead
  ・AQ強度

動画が未選択の場合は、先頭の入力動画を解析します。

解析に使う主な要素:

  ・解像度
  ・フレームレート
  ・ビットレート
  ・4K / 60fps / 120fps などの重さ

便利な使い方:

  1. 動画を追加
  2. 1本選択
  3. 自動走査
  4. 値を確認
  5. 必要なら微調整してエンコード開始

============================================================
🖥️ 7. NVIDIA GPU搭載PCでの動作
============================================================

NVIDIA GPUを搭載し、FFmpeg側で hevc_nvenc が使える場合、このツールの高速処理が最も活きます。

特に有効なモード:

  ✅ 標準：LUT + HEVC/NVENC エンコード
  ✅ LUTのみ適用：品質優先
  ✅ 高速LUT適用：trilinear + NVENC p1
  ✅ 最速LUT適用：nearest + NVENC p1
  ✅ HEVC変換のみ：LUTなし

GeForce RTX 3060 などのNVENC対応GPUなら、CPUエンコードより大幅に高速化しやすいです。

ただし、LUT処理そのものはFFmpegの映像フィルタ処理です。
NVENCは主に「エンコード処理」を高速化します。


============================================================
🟧 8. NVIDIA以外のGPU搭載PCで実行した場合
============================================================

ここは重要です。

このツールの高速エンコード系は NVIDIA NVENC、具体的には FFmpeg の hevc_nvenc を前提にしています。

そのため、以下の環境では挙動が変わります。

  ・AMD Radeon のみ搭載PC
  ・Intel Arc / Intel内蔵GPUのみ搭載PC
  ・NVIDIA GPUがないPC
  ・NVIDIA GPUはあるが、ドライバやFFmpegがNVENC非対応のPC

------------------------------------------------------------
8-1. 何が起きるか ⚠️
------------------------------------------------------------

NVIDIA以外のGPUしかない環境で NVENC モードを実行すると、FFmpeg側でエラーになります。

表示され得る例:

  ・Unknown encoder 'hevc_nvenc'
  ・Cannot load nvcuda.dll
  ・No NVENC capable devices found
  ・ffmpeg が異常終了しました。ExitCode=...

これはツールの故障ではなく、使用しているFFmpegまたはPCに NVENC 実行環境がないことが原因です。

------------------------------------------------------------
8-2. NVIDIA以外で使える可能性がある設定 🧩
------------------------------------------------------------

NVIDIA以外のPCで使う場合は、次の方針になります。

  ✅ 「標準：LUT + HEVC/NVENC エンコード」を選ぶ
  ✅ 「LUT適用 ON」は必要に応じてON
  ✅ 「HEVC NVENC」のチェックをOFF

この場合、内部的には libx265 によるCPUエンコードへ切り替わります。

つまり、NVIDIA GPUがなくても、CPUエンコードとして動作できる可能性があります。
ただし、NVENCよりかなり遅くなります。

------------------------------------------------------------
8-3. 非NVIDIA環境で注意すべきモード 🚫
------------------------------------------------------------

以下のモードは、v1.0.6 の仕様上、NVENCを強く前提にします。

  ・LUTのみ適用：品質優先
  ・高速LUT適用：trilinear + NVENC p1
  ・最速LUT適用：nearest + NVENC p1

これらは「LUT適用系の専用モード」として、HEVC/NVENCを使う設計です。
非NVIDIA環境では失敗する可能性が高いです。

非NVIDIA環境でLUTを適用したい場合は、まず以下を試してください。

  1. 処理モード: 標準：LUT + HEVC/NVENC エンコード
  2. LUT適用 ON: ON
  3. HEVC NVENC: OFF
  4. 10-bit Main10: 必要に応じてON
  5. エンコード開始

これでCPUの libx265 へ回避できます。

------------------------------------------------------------
8-4. AMD / Intel のハードウェアエンコードについて 🧠
------------------------------------------------------------

v1.0.6 は、AMD AMF や Intel QSV を直接選択するGUIにはなっていません。

つまり、以下のようなエンコーダは現時点ではGUIから直接指定しません。

  ・hevc_amf
  ・h264_amf
  ・hevc_qsv
  ・h264_qsv
  ・av1_qsv

将来的に非NVIDIA GPU対応を強化するなら、次期版で以下を追加するのが現実的です。

  ・エンコーダ選択欄
      NVIDIA NVENC
      Intel QSV
      AMD AMF
      CPU libx265

  ・自動GPU検出
  ・NVENC非対応時の自動CPUフォールバック

v1.0.6では、NVIDIA以外なら「標準モード + HEVC NVENC OFF」が安全策です。

🧯 10. よくあるトラブルと対処
============================================================

------------------------------------------------------------
10-1. ffmpeg.exe が見つからない
------------------------------------------------------------

原因:
  ffmpeg.exe のパスが未指定、または間違っています。

対処:
  ・共通設定で ffmpeg.exe を直接指定する
  ・または ffmpeg を PATH に通す
  ・自動検出ボタンを試す

------------------------------------------------------------
10-2. ffprobe.exe が見つからない
------------------------------------------------------------

原因:
  ffprobe.exe のパスが未指定、または間違っています。

対処:
  ・共通設定で ffprobe.exe を直接指定する
  ・自動走査を使う場合は ffprobe.exe が必要

------------------------------------------------------------
10-3. LUTファイルが見つからない
------------------------------------------------------------

原因:
  LUT適用ONなのに .cube ファイルが存在しません。

対処:
  ・LUT .cube 欄に正しいファイルを指定する
  ・ファイル移動後は設定を更新する

------------------------------------------------------------
10-4. NVIDIA以外のPCでエラーになる
------------------------------------------------------------

原因:
  hevc_nvenc が使えない環境で NVENC モードを実行しています。

対処:
  ・標準モードを選ぶ
  ・HEVC NVENC チェックをOFF
  ・CPUエンコードで実行する

------------------------------------------------------------
10-5. 連結後に再生がおかしい
------------------------------------------------------------

原因:
  -c copy 連結に適さない動画が混ざっている可能性があります。

対処:
  ・同じカメラ、同じ設定の動画だけで連結する
  ・.mp4 と .mkv を混在させない
  ・それでもダメなら「互換重視連結」を使う

------------------------------------------------------------
10-6. 画面が狭くてボタンが見えない
------------------------------------------------------------

v1.0.6 は1920×1080 / 125%付近に配慮していますが、表示スケール150%以上や特殊なフォント環境では窮屈になる可能性があります。

対処:
  ・ウィンドウを最大化する
  ・フォーム内のスクロールを使う
  ・Windowsの表示スケールを一時的に100%または125%へ下げる

🛠️ 12. GUIだけでは見えにくい便利テクニック集
============================================================

  💡 テクニック1: ffmpeg / ffprobe 欄は空欄でも使える
      PATH に通っていれば、欄を空にしても ffmpeg / ffprobe が呼ばれます。
      ただし、確実性重視なら直接指定してください。

  💡 テクニック2: 出力先を空欄にすると入力動画と同じフォルダへ出力
      エンコードタブでは、出力先を空欄にした場合、各入力動画のあるフォルダへ出力されます。

  💡 テクニック3: コマンド表示は強力な診断機能
      GUIの設定からFFmpegコマンドを生成し、クリップボードへコピーします。
      コマンドプロンプトで手動実行すれば、FFmpegの詳細エラーを確認しやすくなります。

  💡 テクニック4: 高速LUTは trilinear、最速LUTは nearest 固定
      高速系モードでは補間方式が固定されます。
      選択欄が無効化されても異常ではありません。

  💡 テクニック5: 連結は順番が命
      リストの上から下へ連結されます。
      ファイル名順に追加された後、必要なら「上へ」「下へ」で調整してください。

  💡 テクニック6: 連結で失敗したら互換重視へ切替
      -c copy は速い反面、入力仕様が揃っていることが前提です。
      少しでも怪しい場合は「互換重視連結」を試すと成功しやすくなります。

  💡 テクニック7: まず1本だけで試す
      新しい設定を試す時は、いきなり大量処理せず、短い動画1本で確認してください。
      特にLUTの色味確認では重要です。

  💡 テクニック8: 並列数を上げすぎない
      並列数を上げるほど速くなるとは限りません。
      NVENC、SSD/HDD、CPU、メモリが競合するため、1〜3程度が実用的です。

  💡 テクニック9: Web用に最適化は容量削減ではない
      -movflags +faststart は再生開始位置情報を前方へ移す処理です。
      画質や容量を大きく変えるものではありません。

  💡 テクニック10: エラー相談時は3点セット
      ・画面スクリーンショット
      ・ログ保存したtxt
      ・コマンド表示でコピーしたFFmpegコマンド
      この3つがあると原因特定が非常に早くなります。

🏁 14. まとめ
============================================================

DJI_Action_VideoToolbox_v1.0.6 は、DJI Action 系動画の色変換・HEVC保存・動画連結を1つにまとめた実用ツールです。

基本方針:

  🎨 色を戻すなら LUT適用
  ⚡ 速く処理するなら 高速LUT適用
  🏎️ とにかく確認なら 最速LUT適用
  📦 容量削減なら HEVC変換のみ
  🎞️ クリップ結合なら 動画連結

NVIDIA GPU環境ではNVENCにより高速処理が可能です。
NVIDIA以外の環境では、標準モードで「HEVC NVENC」をOFFにし、CPUエンコードへ逃がすのが安全です。

以上です。快適なDJI動画運用にご活用ください。🚁🎬✨
